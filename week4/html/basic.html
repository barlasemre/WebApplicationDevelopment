Character Entities

Reserved characters in HTML must be replaced with character entities.Characters, not present on your keyboard, can also be replaced by entities.

A character entity looks like this: &entity_name; OR &#entity_number;

If you use the less than (<) or greater than (>) signs in your text, the browser might mix them with tags. To display a less than sign we must write: &lt; or &#60;

Remember that browsers removes extra spaces? To add real spaces to your text, you can use the ∓nbsp; (non-breaking space) character entity.

Links

Use <a> tag to create links. Its href attribute defines the destination of the link. It is possible to open the link in another window or tab by using <a> tag's target attribute. You can write the name of the window or just write "_blank" to open the page in a new window.

You can use links and id attributes together to create bookmarks. Just put an id attribute to any HTML element and use the link as "#id". Look at this documents top menu for an example

Images

You can use <img> tag to place an image into the page. The location of the image must written as the value of the src attribute

Not that <img> tag is an empty element. You don't need to write the end tag, but push "/" character to the end of the start tag

Some browsers may not have the capability to display images or user may close that feature if they want. Or at worse it is possible that image cannot be displayed at all. In these cases browsers uses the alt attribute of the <img> tag to display information about the image. Search engines uses this information to understand more about the image as well.

You can specify the width and the height of the image using width and height attributes. The size is in pixels.

You can use images and links together as well

Forms

HTML forms are used to collect user input. The <form> element defines an HTML form.

The <input> element is the most important form element. The <input> element has many variations, depending on the type attribute: text, password, submit, radio, checkbox, button, color, date, datetime, datetime-local, email, month, number, range, search, tel, time, url, week

The <select> element defines a drop-down list. The options that will be displayed within the select is defined by <option> tags.

The <textarea> element defines a multi-line input field (a text area).

The <button> element defines a clickable button

Whenever a form is submitted, data entered by the user will be sent to the resource specified by the action attribute of the form. The resource can get the values user entered by using the name attribute's value of the input areas. method attribute of the form tells browser which HTTP method should be used while sending data to the resource.

The <fieldset> element groups related data in a form and the <legend> element defines a caption for the fieldset.
